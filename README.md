# local-thread-retrieval

Local-first retrieval and sensemaking system for long-running knowledge work.

## Core capabilities

- read markdown notes and transcript sources (Obsidian/iCloud)
- deterministic retrieval with strict provenance
- thread-based sensemaking
- explicit evidence pinning
- evidence-linked backlog creation
- grounded context assembly for ChatGPT-assisted analysis

## v1 constraints

- read-only source materials
- no silent note mutation
- no external integrations
- no autonomous actions
- no hidden behaviour
- all state changes explicit
- provenance preserved throughout

## Positioning

This system is the local retrieval and evidence substrate for a broader Cognitive Workbench workflow.

It is not:
- an autonomous agent
- a semantic operating system
- a workflow execution engine
- a hidden-memory assistant
